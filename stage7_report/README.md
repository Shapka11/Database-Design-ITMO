## Выбор таблицы для построения BI-метрик

Для построения аналитических дашбордов была выбрана таблица **`public.house_point_logs`**.

### Обоснование выбора

1. **Динамика роста:** Данная таблица представляет собой непрерывный лог событий начисления и вычитания баллов
   факультетов. Она растет значительно быстрее справочников персонажей или факультетов, так как фиксирует каждое
   отдельное действие студентов и преподавателей.
2. **Наличие необходимых полей:** Таблица содержит все типы полей, необходимые для построения полноценных BI-метрик (
   временные метки, категориальные разрезы и числовые показатели для агрегации).

---

### Поля, используемые в метриках

В аналитическом хранилище ClickHouse и дашборде Metabase на основе таблицы `house_point_logs` используются следующие
поля:

| Тип показателя     | Имя поля в БД | Описание                       | Использование в анализе                                                                                       |
|:-------------------|:--------------|:-------------------------------|:--------------------------------------------------------------------------------------------------------------|
| **Временной**      | `created_at`  | Дата и время начисления баллов | Используется для построения временных рядов, анализа активности по дням/часам и выявления трендов.            |
| **Категориальный** | `house_id`    | Идентификатор факультета       | Позволяет сегментировать начисления по факультетам (Гриффиндор, Слизерин и т.д.) для сравнительного анализа.  |
| **Категориальный** | `reason`      | Причина изменения баллов       | Позволяет группировать данные по типам заслуг или нарушений.                                                  |
| **Числовой**       | `points`      | Количество начисленных баллов  | Основная метрика для агрегации: расчет суммы (`SUM`) баллов факультетов, среднего значения (`AVG`) за период. |

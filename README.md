## 📖 Описание предметной области

Система автоматизирует ключевые процессы школы магии:

- 👨‍🎓 управление данными студентов и сотрудников;
- 🪄 учёт освоенных заклинаний и успеваемости;
- 🏆 ведение истории баллов факультетов;
- 🧹 организацию турниров по квиддичу;
- 🚨 регистрацию инцидентов и происшествий;
- 🛡️ контроль безопасности и дисциплины.

---

## ⚙️ Функциональные требования

### 👤 1. Отсутствие дублирования личных данных

Система должна позволять человеку изменять свой статус (например, студент → сотрудник), не создавая новую запись и не требуя повторного ввода базовой информации.

---

### 🏆 2. История баллов факультетов

Необходимо хранить не только текущий счёт факультета, но и полную историю изменений:

- кто начислил или снял баллы;
- какому студенту;
- по какой причине;
- когда произошло изменение.

---

### 📚 3. Трекинг успеваемости

Для каждого студента фиксируется:

- изученное заклинание;
- дата успешного освоения.

---

### 🚨 4. Учёт инцидентов

Система должна вести журнал происшествий:

- один инцидент может включать нескольких участников;
- участники могут иметь разные роли;
- инициатором может выступать конкретный злодей.

---

### 🧹 5. Турнир по квиддичу

Необходимо хранить:

- составы команд с распределением игроков по позициям;
- результаты матчей турнира.

---

## 📏 Бизнес-правила

1. 🦁 Студент распределяется **только в один факультет** на весь период обучения.
2. 🧹 У факультета может существовать **лишь одна активная команда по квиддичу**.
3. 🪄 Волшебная палочка одновременно принадлежит **только одному владельцу**, но владелец может измениться.
4. ⚔️ В матче по квиддичу участвуют **строго две команды**.
5. 👨‍🏫 Изменение баллов факультетов может выполнять **только сотрудник школы**, но не студент.

## 🔄 Миграции

Для управления схемой используются SQL-миграции.

Особенности:

- ✅ каждая миграция идемпотентна;
- ↩️ предусмотрен rollback;
- 🧪 используется тестирование по принципу **staircase** через Seqwall.

### Команды

Накатить одну миграцию:

```bash
scripts/ci-up-one.sh
```

Откатить одну миграцию:

```bash
scripts/ci-down-one.sh
```

### Процесс запуска

При старте контейнера:

1. Выполняются тесты Seqwall на отдельной тестовой базе.
2. При успешном прохождении тестов миграции применяются к основной БД.
3. Применение выполняется до версии, указанной в:

```env
MIGRATION_VERSION
```

# 💾 Резервное копирование

Резервные копии автоматически загружаются в MinIO.

### Настраиваемые параметры

| Параметр | Назначение |
|-----------|------------|
| `BACKUP_INTERVAL` | расписание в формате cron |
| `BACKUP_RETENTION_COUNT` | количество хранимых копий |

### Дополнительный контроль

В Grafana отображаются:

- 🕒 время создания последнего бэкапа;
- 📦 размер последнего бэкапа.

---

# 🔁 Высокая доступность (HA)

Для обеспечения отказоустойчивости PostgreSQL используется следующий стек:

### 🐘 Patroni

Управляет узлами PostgreSQL и координирует репликацию.

### ⚙️ etcd

Используется как распределённое хранилище конфигурации (DCS) и участвует в выборе лидера.

### 🔀 HAProxy

Предоставляет единую точку входа:

- запись направляется на текущий master;
- маршрутизация осуществляется на основе health-check'ов.

---

# ⚡ Оптимизация производительности

Для оценки эффективности оптимизаций применяется инструмент **pgcompare**.

### Используемые артефакты

| Файл | Назначение |
|--------|------------|
| `pgcompare.yaml` | конфигурация |
| `queries_before.sql` | запросы до оптимизации |
| `queries_after.sql` | запросы после оптимизации |
| `report.html` | итоговый отчёт |

### Анализируемые показатели

- 📌 `p95`;
- 📌 `p99`;
- 📌 `QPS`;
- 📌 планы выполнения запросов.
